# O-LoRA: 正交子空间学习用于语言模型持续学习

## 📋 一句话总结

O-LoRA 通过让每个新任务在**与旧任务正交的子空间**中学习，从而在不使用历史数据回放的情况下，有效防止语言模型在学习新任务时遗忘旧知识。

### 第一阶段：准备（Pre-training）

```
│                  预训练语言模型 (PLM)                      │

                           ↓
```

```
│  ┌─────────────┬─────────────┬─────────────┐            │
│  │ Task Def.   │   Options   │    Text     │ → Answer   │
│  │ (任务定义)   │   (选项)     │   (输入文本)  │            │
│  └─────────────┴─────────────┴─────────────┘            │

└─────────────────────────────────────────────────────────┘
                           ↓
```

### 第三阶段：按序学习新任务（Sequential Task Learning）

对于每个新到来的任务 t，执行以下步骤：

```
┌────────────────────────────────────────────────────────────────┐
│                        任务 t 开始学习                           │
│                                                                │
│   ┌──────────────────────────────────────────────────────┐     │
│   │  步骤1: 创建新的LoRA参数                               │     │
│   │                                                      │     │
│   │  A_t ∈ ℝ^(d×r)   ← 新的可训练矩阵 (列向量: a₁ᵗ, a₂ᵗ...aᵣᵗ)  │     │
│   │  B_t ∈ ℝ^(r×k)   ← 新的可训练矩阵 (行向量: b₁ᵗ, b₂ᵗ...bᵣᵗ)  │     │
│   │                                                      │     │
│   │  只在 W_q (Query) 和 W_v (Value) 注意力权重上应用      │     │
│   └──────────────────────────────────────────────────────┘     │
│                           ↓                                    │
│   ┌──────────────────────────────────────────────────────┐     │
│   │  步骤2: 正交约束                                       │     │
│   │                                                      │     │
│   │  要求: A_t 的列向量 与 所有旧任务 A_i 的列向量正交        │     │
│   │                                                      │     │
│   │  即: A_i^T × A_t = 0  (对于所有 i < t)                │     │
│   └──────────────────────────────────────────────────────┘     │
│                           ↓                                    │
│   ┌──────────────────────────────────────────────────────┐     │
│   │  步骤3: 固定旧任务参数                                 │     │
│   │                                                      │     │
│   │  旧任务 LoRA 参数 {A_i, B_i | i < t} 保持冻结         │     │
│   │  只更新当前任务的 {A_t, B_t}                          │     │
│   └──────────────────────────────────────────────────────┘     │
│                                                                │
└────────────────────────────────────────────────────────────────┘
                           ↓
```

```
│                                                          │
│  推理时无需任务ID，直接使用合并后的模型                     │
└─────────────────────────────────────────────────────────┘
```

## 📐 核心公式详解

### 公式1：LoRA的前向传播

$$h = W_{init}x + \Delta Wx = W_{init}x + ABx$$

| 符号 | 含义 |
|------|------|
| $h$ | 层的输出向量 |
| $W_{init}$ | 预训练模型的原始权重矩阵（固定不动） |
| $x$ | 输入向量 |
| $\Delta W = AB$ | LoRA新增的低秩更新 |
| $A \in \mathbb{R}^{d \times r}$ | LoRA的左矩阵（d=输入维度, r=秩） |
| $B \in \mathbb{R}^{r \times k}$ | LoRA的右矩阵（k=输出维度） |
| $r$ | **秩（Rank）**，远小于 min(d,k)，典型值如2,4,8,16 |

**通俗理解**：LoRA把权重更新拆成两个小矩阵A和B的乘积，这样参数量从d×k大幅降到(d+k)×r。

---

### 公式2：任务t的LoRA子空间定义

$$A_t = [a_1^t, a_2^t, ..., a_r^t]$$

$$U_t = span\{a_1^t, a_2^t, ..., a_r^t\}$$

| 符号 | 含义 |
|------|------|
| $A_t$ | 任务t的LoRA左矩阵 |
| $a_i^t$ | $A_t$的第i个列向量 |
| $U_t$ | 任务t的**参数更新子空间**（由A_t的列向量张成） |

**通俗理解**：每个任务的LoRA参数定义了一个"学习方向"（子空间），模型参数的更新主要在这个方向上进行。

---

### 公式3：正交性条件

$$O_{i,t} = A_i^T A_t = 0$$

| 符号 | 含义 |
|------|------|
| $O_{i,t}$ | 任务i和任务t的交叉矩阵 |
| $A_i^T$ | 任务i的LoRA左矩阵的转置 |
| $A_t$ | 任务t的LoRA左矩阵 |

**通俗理解**：让旧任务的"学习方向"和新任务的"学习方向"垂直（正交），这样学习新任务时就不会"踩到"旧任务的学习痕迹。

---

### 公式4：总体训练目标

$$\mathcal{L} = \sum_{x,y \in D_t} \log p_\Theta(y|x) + \lambda_1 \sum_{i=1}^{t-1} \mathcal{L}_{orth}(A_i, A_t)$$

| 符号 | 含义 |
|------|------|
| $\sum_{x,y \in D_t} \log p_\Theta(y \mid x)$ | **任务损失**：在新任务数据上的对数似然 |
| $\lambda_1$ | 正交损失的权重（超参数） |
| $\mathcal{L}_{orth}(A_i, A_t)$ | **正交损失**：衡量新旧任务子空间的正交程度 |

**通俗理解**：训练目标 = 学好新任务 + 不要干扰旧任务

---

### 公式5：正交损失的具体形式

$$\mathcal{L}_{\mathrm{orth}}(A_i, A_t) = \sum_{j,k} |O_{i,t}[j,k]|^2$$

| 符号 | 含义 |
|------|------|
| $O_{i,t}[j,k]$ | 矩阵$O_{i,t}$的第j行第k列元素 |
| $\sum_{j,k}$ | 对所有矩阵元素求和 |

**通俗理解**：让新旧任务的A矩阵乘积矩阵的所有元素都接近0（最小二乘法）。

---

### 公式6：推理时合并LoRA参数

$$W_{init} := W_{init} + \sum_{i=1}^{t} A_i B_i$$

| 符号 | 含义 |
|------|------|
| $W_{init}$ | 原始预训练权重 |
| $A_i B_i$ | 第i个任务的LoRA更新 |
| $\sum_{i=1}^{t}$ | 将所有任务的LoRA更新累加 |

**通俗理解**：训练时用多个LoRA分支，推理时把它们"消化"成一个统一的权重矩阵，这样推理时不需要管理多个LoRA。

**为什么这样做能防止遗忘？**

1. **物理类比**：想象你在一个有很多脚印的沙滩上走路
   - 传统方法：新脚印会踩在旧脚印上，把它们弄模糊（遗忘）
   - O-LoRA：找到一块没有脚印的空地走（新方向的垂直方向）

2. **数学保证**：如果两个向量正交，它们的点积为0
   - 梯度正交 = 一个任务参数更新对另一个任务的损失影响最小
   - 学习新任务时，几乎不会改变旧任务的损失函数值

**列的含义详解：**

| 列名 | 含义 | 解释 |
|------|------|------|
| **RF** (Rehearsal-Free) | 无需数据回放 | 不需要存储历史任务的数据 |
| **PE** (Parameter-Efficient) | 参数高效 | 只需要训练少量额外参数 |
| **TIF** (Task-ID during Inference) | 推理时需要任务ID | 测试时需要知道样本属于哪个任务才能正确预测 |
| **UT** (Unseen Tasks) | 支持未见任务 | 可以处理训练时没有见过的任务类型 |

**逐行分析各方法：**

```
EWC (Elastic Weight Consolidation):
├─ RF: ✓ 不需要数据回放
├─ PE: ✗ 需要训练整个模型的所有参数
├─ TIF: ✗ 推理时需要任务ID
└─ UT: ✗ 无法泛化到未见任务
原因: EWC在所有任务上共享整个模型参数

A-GEM (Average Gradient Episodic Memory):
├─ RF: ✓ 不需要数据回放
├─ PE: ✗ 需要训练整个模型
├─ TIF: ✓ 不需要任务ID
└─ UT: ✗ 无法泛化到未见任务
特点: 使用记忆缓冲区存储梯度，但不是数据

MBPA++ (Memory-Based Parameter Adaptation):
├─ RF: ✓ 不需要数据回放
├─ PE: ✗ 需要训练整个模型
└─ 主要是基于记忆的快速适配

L2P (Learn to Prompt):
├─ RF: ✓ 不需要数据回放
├─ PE: ✓ 只训练prompt
├─ TIF: ✓ 不需要任务ID
└─ UT: ✓ 使用prompt引导泛化
特点: 用prompt pool管理任务

LFPT5 (Lifelong Few-shot Prompt Tuning):
├─ RF: ✓ 不需要数据回放
├─ PE: ✓ 只训练soft prompt
└─ 只能用于T5模型
```

**O-LoRA的独特优势（四个✓全满）：**

```
O-LoRA:
├─ RF: ✓ 不需要存储历史数据
│    原因: 用LoRA参数本身近似历史任务的梯度子空间
│    
├─ PE: ✓ 每个任务只增加 O((d+k)×r) 个参数
│    对比: EWC需要训练 d×k 个参数
│    
├─ TIF: ✓ 推理时不需要任务ID
│    原因: 可以合并所有LoRA到原始权重
│    
└─ UT: ✓ 可以处理未见任务
     原因: 使用指令微调，保留泛化能力
```

### 训练循环（第10-29行）


```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    第14-15行: 计算任务损失                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【代码】 L_task = Σ_{(x,y)∈B} log p_Θ(y|x)                                │
│                                                                             │
│  【含义】 计算当前批次中，模型预测正确的对数似然                               │
│           log p_Θ(y|x) 表示: 给定输入x，模型预测为y的概率的对数               │
│           Σ表示对批次中所有样本求和                                           │
│                                                                             │
│  【分解】                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   p_Θ(y|x) 是什么意思？                                              │   │
│  │   ├── x = "这部电影太棒了!"                                           │   │
│  │   ├── y = "正面"                                                     │   │
│  │   └── p_Θ("正面"|"这部电影太棒了!") = 模型认为这是正面的概率          │   │
│  │       = 0.92 (92%)                                                   │   │
│  │                                                                      │   │
│  │   log p_Θ(y|x) 是什么意思？                                          │   │
│  │   ├── log(0.92) ≈ -0.0836                                           │   │
│  │   └── 用对数是为了方便计算(连乘变加法)                                │   │
│  │                                                                      │   │
│  │   L_task = 最大化这个值 = 最小化负的这个值                            │   │
│  │           = 让模型越来越准确地预测                                    │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【比喻】 想象你在做选择题测验                                               │
│           L_task就是你做对的题目数量的"得分"                                  │
│           做对越多题，L_task越高                                            │
│                                                                             │
│  【实际例子】                                                                │
│  批次B = [样本1, 样本2, 样本3]                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 样本1: "这部电影太棒了!" → 预测"正面" → 真实"正面" → log p = -0.05   │   │
│  │ 样本2: "非常无聊的片子" → 预测"负面" → 真实"负面" → log p = -0.02   │   │
│  │ 样本3: "还行吧" → 预测"正面" → 真实"中性" → log p = -1.20           │   │
│  │                                                                      │   │
│  │ L_task = (-0.05) + (-0.02) + (-1.20) = -1.27                        │   │
│  │ (负数是正常的，因为我们是最小化损失函数)                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    第17-22行: 计算正交损失                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【代码】                                                                   │
│  L_orth = 0                                                                 │
│  for i = 1 to t-1 do                                                        │
│      O_{i,t} = Aᵢ^T × Aₜ                                                   │
│      L_orth += Σ_{j,k} ||O_{i,t}[j,k]||²                                   │
│  end for                                                                    │
│                                                                             │
│  【含义】 让当前任务t的LoRA矩阵Aₜ与所有旧任务i的LoRA矩阵Aᵢ保持正交             │
│                                                                             │
│  【逐步分解】                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │  步骤1: O_{i,t} = Aᵢ^T × Aₜ                                         │   │
│  │  ├─ Aᵢ^T: 任务i的LoRA矩阵的转置 (r × d)                             │   │
│  │  ├─ Aₜ: 任务t的LoRA矩阵 (d × r)                                     │   │
│  │  └─ O_{i,t}: 两个矩阵的乘积 (r × r)                                  │   │
│  │                                                                      │   │
│  │  步骤2: L_orth += Σ_{j,k} ||O_{i,t}[j,k]||²                          │   │
│  │  ├─ 对O_{i,t}中的每个元素求平方                                      │   │
│  │  └─ 求和得到一个标量                                                  │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【为什么这样可以保证正交？】                                                │
│                                                                             │
│  假设有两个向量 a 和 b, 它们正交 ⟺ a^T × b = 0                              │
│                                                                             │
│  我们的目标是让 Aᵢ 和 Aₜ 的列向量都互相正交                                   │
│  即: 对于Aᵢ的每一列 a_j 和 Aₜ的每一列 b_k, 有 a_j^T × b_k = 0                │
│                                                                             │
│  而 Aᵢ^T × Aₜ 的第(j,k)个元素正是 a_j^T × b_k                               │
│  所以让 Aᵢ^T × Aₜ = 0 就是让所有列向量对正交!                                │
│                                                                             │
│  【比喻】 想象你在建房子                                                      │
│           旧任务是已有的四面墙(Aᵢ的列向量)                                    │
│           新任务是新建的墙(Aₜ的列向量)                                        │
│           你需要确保新墙与所有旧墙都垂直                                      │
│           O_{i,t} = Aᵢ^T × Aₜ 就是测量"墙之间的角度"                        │
│           L_orth = Σ||O_{i,t}||² 就是"不垂直的程度"                         │
│           最小化L_orth就是让所有墙都垂直                                      │
│                                                                             │
│  【图示】                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   假设 r=2 (两个列向量), 那么 O_{i,t} 是一个 2×2 矩阵:               │   │
│  │                                                                      │   │
│  │        ┌─────────────────────────────────────────────────────────┐  │   │
│  │        │                    Aᵢ^T × Aₜ = O_{i,t}                  │  │   │
│  │        │  ┌───────────────────────────────────────────────────┐   │  │   │
│  │        │  │ O[1,1] = a_1 · b_1  │  O[1,2] = a_1 · b_2        │   │  │   │
│  │        │  ├───────────────────────────────────────────────────┤   │  │   │
│  │        │  │ O[2,1] = a_2 · b_1  │  O[2,2] = a_2 · b_2        │   │  │
│  │        │  └───────────────────────────────────────────────────┘   │  │   │
│  │        └─────────────────────────────────────────────────────────┘  │   │
│  │                                                                      │   │
│  │   目标: 让所有 O[j,k] ≈ 0                                            │   │
│  │   即: a_1 ⊥ b_1, a_1 ⊥ b_2, a_2 ⊥ b_1, a_2 ⊥ b_2                    │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【实际例子】                                                                │
│  假设 t=3 (正在学习第三个任务), 需要与 i=1,2 两个旧任务正交                    │
│                                                                             │
│  与任务1的正交损失:                                                          │
│  O_{1,3} = A₁^T × A₃                                                       │
│  假设 O_{1,3} = [[0.1, 0.05], [0.02, 0.08]]                                │
│  L_orth_1 = 0.1² + 0.05² + 0.02² + 0.08² = 0.0193                         │
│                                                                             │
│  与任务2的正交损失:                                                          │
│  O_{2,3} = A₂^T × A₃                                                       │
│  假设 O_{2,3} = [[0.03, 0.01], [0.06, 0.04]]                               │
│  L_orth_2 = 0.03² + 0.01² + 0.06² + 0.04² = 0.0062                        │
│                                                                             │
│  L_orth = L_orth_1 + L_orth_2 = 0.0255                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    第24-25行: 总损失 = 任务损失 + 正交损失                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【代码】 L_total = L_task + λ₁ × L_orth                                    │
│                                                                             │
│  【含义】 总损失 = 学会当前任务 + 不忘记旧任务                                 │
│                                                                             │
│  【分解】                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   L_total = L_task + λ₁ × L_orth                                    │   │
│  │                                                                      │   │
│  │        │           │          │                                      │   │
│  │        │           │          └─ 权重,控制正交约束的重要性             │   │
│  │        │           │               λ₁越大 → 越强调正交                │   │
│  │        │           │               典型值: λ₁ = 0.5                 │   │
│  │        │           │                                                       │   │
│  │        │           └─ 正交损失                                          │   │
│  │        │               衡量与旧任务的正交程度                           │   │
│  │        │               L_orth越大 → 与旧任务越不正交                    │   │
│  │        │                                                           │   │
│  │        └─ 任务损失                                                   │   │
│  │            衡量当前任务的预测准确度                                   │   │
│  │            L_task越大(负值越小) → 预测越准确                          │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【比喻】 想象你在练习乐器                                                   │
│           L_task = 演奏新曲子的准确度                                       │
│           L_orth = 不要打扰邻居的安静程度                                    │
│           λ₁ = 邻居投诉的严重程度                                           │
│                                                                             │
│           如果λ₁很高，你可能会牺牲一些演奏技巧来保持安静                       │
│           如果λ₁很低，你可能会吵到邻居                                        │
│                                                                             │
│  【实际例子】                                                                │
│  假设当前批次:                                                               │
│  L_task = -1.27 (当前任务预测不太准确)                                      │
│  L_orth = 0.0255 (与旧任务有轻微不正交)                                     │
│  λ₁ = 0.5                                                                   │
│                                                                             │
│  L_total = (-1.27) + 0.5 × 0.0255                                          │
│          = -1.27 + 0.0128                                                  │
│          = -1.2572                                                         │
│                                                                             │
│  优化目标: 最小化 L_total                                                   │
│  → 需要同时: (1)降低L_task (提高预测准确度)                                  │
│            (2)降低L_orth (保持与旧任务正交)                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    第27-28行: 反向传播更新参数                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【代码】 更新 Θ 以最小化 L_total                                            │
│                                                                             │
│  【含义】 用梯度下降法更新模型参数                                            │
│           但注意! 只有 Aₜ 和 Bₜ 会被更新                                      │
│           预训练模型参数 Θ_base 和旧的 LoRA (Aᵢ, Bᵢ) 保持不变                │
│                                                                             │
│  【梯度流动示意】                                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   L_total                                                           │   │
│  │       │                                                             │   │
│  │       ├──→ L_task ──→ Θ (预训练模型) ──→ 但梯度被阻断!              │   │
│  │       │                                                             │   │
│  │       │                    因为LoRA的更新路径:                        │   │
│  │       │                    Θ包含W_q, W_v等注意力权重                 │   │
│  │       │                    但ΔW_q = AₜBₜ (只有当前任务的!)          │   │
│  │       │                                                             │   │
│  │       └──→ L_orth ──→ Aₜ, Bₜ ← 只有这里接收梯度!                     │   │
│  │                            (Aᵢ, Bᵢ 被冻结, 无梯度)                   │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【比喻】 想象你在装修房间                                                   │
│           L_total = 装修总费用                                               │
│           L_task = 买新家具(当前任务)的费用                                  │
│           L_orth = 隔音材料的费用(保护旧任务)                                │
│                                                                             │
│           你只能买新家具(Aₜ, Bₜ)                                             │
│           旧家具(Aᵢ, Bᵢ)已经定好了,不能换                                    │
│           预装修的房间结构(预训练模型)也不能动                                │
│                                                                             │
│  【实际更新过程】                                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   假设初始: Aₜ = [[0.1, 0.2], [0.3, 0.4]]                           │   │
│  │            Bₜ = [[0.5, 0.6], [0.7, 0.8]]                            │   │
│  │                                                                      │   │
│  │   计算梯度: ∂L_total/∂Aₜ, ∂L_total/∂Bₜ                              │   │
│  │                                                                      │   │
│  │   假设梯度: ∂L_total/∂Aₜ = [[0.01, -0.01], [0.02, -0.02]]            │   │
│  │            学习率 η = 0.001                                          │   │
│  │                                                                      │   │
│  │   更新: Aₜ_new = Aₜ - η × ∂L_total/∂Aₜ                              │   │
│  │        = [[0.1, 0.2], [0.3, 0.4]] - 0.001 × [[0.01, -0.01], ...]   │   │
│  │        ≈ [[0.09999, 0.20001], [0.29998, 0.40002]]                   │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└───────────────────────────────────────────────────────────────��─────────────┘
```

## 为什么需要测试多个Order？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        为什么测试不同任务顺序？                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【核心问题】                                                                │
│                                                                             │
│  在持续学习中，任务A先学和任务B先学，效果一样吗？                            │
│  答案是: 不一样!                                                             │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │  场景1: 先学"简单任务"，再学"困难任务"                               │   │
│  │  ├─ 简单任务打好基础                                                  │   │
│  │  ├─ 模型可能获得"通用能力"                                            │   │
│  │  └─ 但后期可能遗忘简单任务的基础知识                                    │   │
│  │                                                                      │   │
│  │  场景2: 先学"困难任务"，再学"简单任务"                                │   │
│  │  ├─ 困难任务可能占用更多"容量"                                        │   │
│  │  ├─ 简单任务可能更容易学会                                             │   │
│  │  └─ 但可能受困难任务的影响，产生干扰                                    │   │
│  │                                                                      │   │
│  │  场景3: 相似的任务挨着学 vs 分开学                                     │   │
│  │  ├─ 相似任务挨着学: 可能产生正向迁移(有帮助)                           │   │
│  │  └─ 相似任务分开学: 可能产生负向迁移(有干扰)                            │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【实验设计】                                                               │
│                                                                             │
│  为了公平评估CL方法，需要测试多种任务顺序:                                    │
│  ├─ 如果一个方法在所有顺序下都好 → 真正有效                                  │

```

```
│                        任务顺序影响的直观例子                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【假设场景: 教一个AI学习三种技能】                                           │
│                                                                             │
│  技能A: 识别猫狗 (图像分类)                                                 │

│  技能C: 识别鸟类 (图像分类)                                                │
│                                                                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━    │
│                                                                             │
│  场景1: A → B → C (猫狗 → 苹果橘子 → 鸟类)                                 │
│  ├─ 先学猫狗: 建立"动物分类"的基本概念                                       │

│                                                                             │

│  ├─ 先学苹果橘子: 建立"水果"概念                                             │
│  ├─ 再学鸟类: 新概念"动物"，与水果不同                                      │
│  └─ 最后学猫狗: 又一种"动物"，与鸟类类似                                      │
│  优点: 先学简单(水果)，再学复杂(动物)                                       │
│  缺点: 学猫狗时可能受鸟类影响                                                │
│                                                                             │

│  关键洞察:                                                                  │

│                                                                             │

```

## 什么时候用 ROUGE？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        什么时候用 ROUGE？                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ROUGE 用于【文本生成任务】，例如:                                          │
│                                                                             │
│  1. 文本摘要                                                                │
│     输入: 一篇新闻文章                                                       │
│     生成: "今日股市上涨5%，创历史新高..."                                    │
│     参考: "沪深两市今日大幅上涨..."                                          │
│     → 计算生成的摘要和参考摘要的n-gram重叠                                   │
│                                                                             │
│  2. 机器翻译                                                                │
│     输入: "Hello, world!"                                                  │
│     生成: "你好，世界！"                                                    │
│     参考: "你好，世界！"                                                    │
│     → 计算翻译结果和参考翻译的重叠                                           │
│                                                                             │
│  3. 对话生成                                                                │
│     输入: "今天天气真好"                                                    │
│     生成: "是啊，适合出去游玩"                                              │
│     参考: "确实，阳光明媚"                                                  │
│     → 计算生成回复和参考回复的重叠                                           │
│                                                                             │
│  【这篇论文为什么不用ROUGE】                                                │
│  因为标准CL基准的任务都是分类任务，不是生成任务                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 总结

| 问题 | 答案 |
|------|------|
| Table 2 用什么指标？ | **accuracy**（分类准确率） |
| 用 ROUGE 吗？ | **不用**，这是分类任务，不是生成任务 |
| accuracy 怎么算？ | 预测正确的样本数 / 总样本数 |
| 这个指标代表什么？ | 模型对每个任务分类正确的能力 |
| 为什么是"平均"？ | 学完所有任务后，分别测每个任务的准确率，再取平均 |
